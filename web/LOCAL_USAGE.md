## 🔧 配置说明

### 输出目录位置

默认输出目录：`./outputs`

**查看完整路径**:
```bash
# 在项目根目录运行
cd outputs
pwd  # Linux/Mac
cd   # Windows
```

**修改输出目录**:

编辑 `api/app.py`:

```python
# 找到这一行
OUTPUT_DIR = Path("./outputs")

# 改为你想要的路径，例如：
OUTPUT_DIR = Path("D:/WatermarkResults")  # Windows
OUTPUT_DIR = Path("/Users/你的用户名/Pictures/Results")  # macOS
OUTPUT_DIR = Path("/home/你的用户名/results")  # Linux
```

### 文件命名规则

输出文件命名格式：`{task_id}_result.{ext}`

例如：
- `a1b2c3d4-e5f6-..._result.png`
- `x9y8z7w6-v5u4-..._result.mp4`

**task_id** 是唯一标识符，保证文件名不重复。

## ⚠️ 注意事项

### 1. 权限问题

如果打开文件夹失败，可能是权限问题：

**解决方案**:
- 确保有读写输出目录的权限
- Windows: 右键 → 属性 → 安全
- Linux/Mac: `chmod 755 outputs`

### 2. 路径包含中文

如果路径包含中文可能导致问题：

**解决方案**:
- 使用英文路径
- 或更新 Python 版本到 3.7+

### 3. 文件管理器未打开

如果点击按钮后没反应：

**排查**:
1. 查看浏览器控制台（F12）
2. 查看服务器日志
3. 检查文件是否确实生成

**手动打开**:
- Windows: 在地址栏输入 `.\outputs`
- macOS: Finder → 前往 → 前往文件夹 → 输入路径
- Linux: 打开文件管理器，导航到项目的 outputs 目录

## 📊 功能对比

### 下载 vs 打开文件夹

| 功能 | 下载 | 打开文件夹 |
|------|------|-----------|
| **操作** | 点击下载按钮 | 点击打开文件夹 |
| **结果** | 保存到默认下载目录 | 打开输出目录 |
| **优势** | 方便归档 | 快速查看 ⭐ |
| **选择** | 需要选择保存位置 | 直接打开 ⭐ |
| **适用** | 需要移动文件 | 本地使用 ⭐ |

**推荐**:
- 本地使用：优先使用"打开文件夹" ⭐
- 需要归档：使用"下载"功能

## 🐛 故障排除

### Q1: 点击"打开文件夹"没反应？

**检查**:
1. 浏览器控制台是否有错误
2. 服务器日志是否有报错
3. 输出目录是否存在

**解决**:
```bash
# 手动创建输出目录
mkdir outputs

# 检查权限
chmod 755 outputs  # Linux/Mac
```

### Q2: 文件夹打开了但找不到文件？

**原因**: 文件可能还未完成

**解决**:
- 等待处理完成
- 查看任务状态
- 刷新文件管理器

### Q3: Linux 下无法打开文件夹？

**原因**: 未安装 xdg-open

**解决**:
```bash
sudo apt-get install xdg-utils  # Ubuntu/Debian
sudo yum install xdg-utils      # CentOS/RHEL
```
